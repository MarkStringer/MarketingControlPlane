# Software as an Escape Attempt

## Summary
This video argues that software needs to “escape” into real use as early as possible, because only working software reveals what will actually stop release, how users really respond, and whether the product has genuine pull. It also argues that many people connected to a project may resist release because once software is in users’ hands, the power dynamics of the project change.

## Key ideas
- Software has to be released into real use to reveal hidden blockers and discover real value.
- Once software reaches users, the project can shift from internal push to external pull.
- Clients, users, and developers may all resist release, but often for different reasons.
- Working software is experienced very differently from a feature list, a specification, or a design.
- Velocity should be understood as movement in several dimensions, such as toward live data, production environments, and genuine use, not just more functionality.

## Notable phrases
- software needs to escape
- working software is magical
- the power dynamic of the project is completely different
- velocity is speed in a bunch of different directions
- almost nobody will want you to do that
