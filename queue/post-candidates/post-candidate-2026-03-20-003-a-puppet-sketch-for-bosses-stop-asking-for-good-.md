# A puppet sketch for bosses: Stop asking for good news from a real project

## Hook

Wild idea for a short video: a boss, a project manager, and a puppet arguing about whether bad news is failure or the first useful signal that the project is real.

## Draft

Possible short script / post idea:

Boss: "Why am I hearing problems already?"
Puppet: "Because the project exists."
Boss: "I wanted confidence."
Puppet: "Then buy a forecast. If you want delivery, listen to the data."

Under the joke is a serious point.

If a software project is really happening, it will produce surprises, friction, and bad news.
That is not a sign that reality has gone wrong.
It is a sign that reality has arrived.

The dangerous move is when hierarchy trains people to clean up the message on the way up.
Then the boss gets theatre instead of evidence.

Better leadership means creating the kind of conversation where awkward signals can stay intact long enough to become action.
Not public blame. Not silence. Actual discussion.

That is one reason I keep coming back to the idea that bad news is data.

A project manager helps make the project possible by surfacing delivery problems.
A product owner helps make it valuable by interpreting what users actually want.
If bosses can hear both clearly, they give the project a fighting chance.

This could work as a whacky video/dialogue post, but the underlying point is very practical: don’t ask your project to sound healthy. Ask it to tell you what reality is saying.

It’s a theme I explore in Delivering the Impossible.
