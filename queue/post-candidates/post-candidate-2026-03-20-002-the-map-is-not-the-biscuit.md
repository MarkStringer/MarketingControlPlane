# The map is not the biscuit

## Draft

A specification is not software in the same way a biscuit tin is not a biscuit.

Projects go wrong when people expect an exact match between the idea, the description, and the thing in the world.

Better delivery starts when we trade “all / same / exact match” thinking for:
- some value
- some evidence
- some learning
- some working software

The project is a bet.
Not a vending machine.
