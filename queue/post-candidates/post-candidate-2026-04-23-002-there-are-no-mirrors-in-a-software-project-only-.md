## Draft

There are no mirrors in a software project. Only prisms.

Every stakeholder — the boss, the PM, the dev, the user — is looking at the same project and seeing something genuinely different.

Not wrong. Different.

The boss sees a commitment they made to the board.
The developer sees a system that will take three times longer than anyone thinks.
The user sees a problem that nobody has quite named correctly yet.

None of them are lying. They're all refracting.

The mistake is assuming everyone's prism works the same way — that if you just communicate clearly enough, everyone will converge on the same picture.

They won't. And that's not a failure of communication. It's the nature of the thing.

The skill is learning to hold multiple refractions at once. To ask: what is *this person's* distortion telling me that I can't see from where I'm standing?

Point of view is worth 80 IQ points. The question is whose you're borrowing.
