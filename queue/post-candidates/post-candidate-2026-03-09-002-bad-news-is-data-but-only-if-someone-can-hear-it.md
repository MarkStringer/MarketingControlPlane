# Bad news is data, but only if someone can hear it

## Hook

If a project is real, it will produce bad news. The question is whether your organisation can convert that into action.

## Rationale

## Draft

If a software project is real, it will generate surprises, friction, and bad news.

That isn’t failure.
It’s evidence.

The bigger danger is when hierarchy turns the evidence into silence, spin, or delay.

Bad news is data.
But data only helps if someone can turn it into action.

Good project leadership is not "protecting stakeholders from bad news".
It is creating low-distortion conversations where reality can be discussed early enough to matter.

## Why now

Useful whenever delivery teams are under pressure, especially in environments where status reporting tends to hide problems until they are expensive. It also reinforces a signature phrase from the message house.

## Supporting ideas

- Real projects generate problems, surprises, and bad news.
- Bad news should be seen as data, not just embarrassment or failure.
- Hierarchy distorts communication and causes bad news to be softened or hidden.
- Leaders need honest, low-distortion conversations that turn evidence into action.
