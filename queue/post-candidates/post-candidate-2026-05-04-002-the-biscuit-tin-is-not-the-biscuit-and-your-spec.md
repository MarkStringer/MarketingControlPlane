# The biscuit tin is not the biscuit — and your spec is not the software

## Hook

A picture of biscuits on a tin is not the same as biscuits. Your 200-page spec is not the same as working software. We know this. We act like we don't.

## Draft

A picture of biscuits on a tin is not biscuits.

You know this. I know this. Everyone in the supermarket knows this.

And yet.

Every week, somewhere, a stakeholder looks at a wireframe, a document, a diagram — and starts acting as if the product already exists. As if the only thing left is pressing a button.

This is exact-match fantasy: the belief that a good enough description is basically the same as the real thing.

It isn't.

The description has no bugs. It runs at infinite speed. It fits every user perfectly. It never crashes. It costs nothing to change.

The software, meanwhile, is in the world. The world is messy.

The gap between the tin and the biscuit is where almost all project pain lives.

Working software — actual, running, incomplete, imperfect working software — is the only thing that collapses that gap. Nothing else does.

Stop selling the tin. Ship the biscuit.
