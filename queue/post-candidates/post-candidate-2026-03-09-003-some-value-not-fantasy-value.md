# Some value, not fantasy value

## Hook

A lot of delivery pain comes from one habit: treating the original idea as if it already exists in perfect form.

## Rationale

This gives a restrained book-aware post that directly supports awareness goals while still delivering a useful idea. It ties together several distinctive arguments from the manuscript: asymmetry, exact-match fantasy, and projects as bets.

## Draft

One theme running through *Delivering the Impossible* is that projects become more manageable when we stop thinking in terms like all, same, forever, and exact match.

That style of thinking sounds strong, but it causes a lot of avoidable pain:
- wanting everything
- resisting prioritisation
- treating descriptions as if they were the product
- acting as if a promise can remove uncertainty

A better discipline is asymmetrical thinking:
- some, not all
- useful now, not perfect later
- working software, not just description
- adaptation, not fantasy certainty

This is not an argument against standards or commitment.
It’s an argument for a better balance.

Projects are bets.
They get more deliverable when we narrow the bet, test it in reality, and learn from what comes back.

That is a big part of what the book is trying to do: replace instinctive but misleading ways of seeing projects with better ones.

## Supporting ideas

- Symmetrical thinking uses all/same/forever/exact match and creates unrealistic expectations.
- Asymmetrical thinking accepts some value, different viewpoints, change, and imperfect matches.
- Working software helps shift people away from exact-match fantasy.
- The book’s conclusion argues for better ways of seeing projects and describes projects as more deliverable when teams focus on some value rather than all value.
